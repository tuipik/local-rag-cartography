# WORKFLOW

Для кожного етапу використовується наступний цикл:

1. Планування.
2. Обговорення альтернатив.
3. Реалізація.
4. Рев'ю.
5. Приймання.
6. Оновлення документації.
7. Перехід до наступного етапу.

Етап не вважається завершеним без рев'ю та оновлення документації.

Важливі архітектурні рішення повинні бути внесені в DECISIONS.md.

## Перед chunking

Перед реалізацією chunking обов'язково виконується аналіз корпусу документів.

Chunking strategy не обирається наперед. Вона визначається на основі фактичної структури документів.
